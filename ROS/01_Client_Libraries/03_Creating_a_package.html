https://docs.ros.org/en/iron/Tutorials/Beginner-Client-Libraries/Creating-Your-First-ROS2-Package.html <패키지란 무엇인가?> 패키지는 ROS 2 코드를 위한 조직적인 단위입니다. 코드를 설치하거나 다른 사람과 공유하고 싶다면, 패키지로 조직화할 필요가 있습니다. 패키지를 사용하면 ROS 2 작업을 릴리스하고 다른 사람들이 쉽게 빌드하고 사용할 수 있습니다. ROS 2에서 패키지 생성은 빌드 시스템으로 ament를 사용하고 빌드 도구로 colcon을 사용합니다. 공식적으로 지원되는 CMake 또는 Python을 사용하여 패키지를 생성할 수 있으며, 다른 빌드 유형도 존재합니다. 패키지 내 코드를 빌드하는 방법을 설명하는 CMakeLists.txt 파일 패키지의 공개 헤더가 포함된 include/ 디렉토리 패키지에 대한 메타 정보를 담고 있는 package.xml 파일 패키지의 소스 코드가 포함된 src 디렉토리 가장 간단한 패키지의 파일 구조는 다음과 같을 수 있습니다 my_package/ CMakeLists.txt include/my_package/ package.xml src/ 하나의 작업 공간에 원하는 만큼의 패키지를 각자의 폴더에 넣을 수 있습니다. 하나의 작업 공간에 다른 빌드 유형의 패키지들(CMake, Python 등)을 가질 수도 있습니다. 중첩된 패키지는 가질 수 없습니다. 최상위 작업 공간을 "깨끗하게" 유지하기 위해 작업 공간 내에 src 폴더를 두고, 그 안에 패키지를 생성하는 것이 좋은 방법입니다. 간단한 작업 공간은 다음과 같을 수 있습니다. workspace_folder/ src/ cpp_package_1/ CMakeLists.txt include/cpp_package_1/ package.xml src/ py_package_1/ package.xml resource/py_package_1 setup.cfg setup.py py_package_1/ ... cpp_package_n/ CMakeLists.txt include/cpp_package_n/ package.xml src/ 필수 조건 이전 튜토리얼의 지침을 따라 ROS 2 작업 공간을 가지고 있어야 합니다. 이 작업 공간에서 패키지를 생성할 것입니다. <1. 패키지 생성> 이전 튜토리얼에서 만든 작업 공간인 ros2_ws를 새 패키지에 사용합시다. 패키지 생성 명령을 실행하기 전에 src 폴더에 있는지 확인하세요. cd ~/ros2_ws/src ROS 2에서 새 패키지를 생성하는 명령 구문은 다음과 같습니다. ros2 pkg create --build-type ament_cmake 이 튜토리얼에서는 선택적 인자인 --node-name과 --license를 사용할 것입니다. --node-name 옵션은 패키지에 간단한 Hello World 유형의 실행 파일을 생성하고, --license는 패키지의 라이센스 정보를 선언합니다. 터미널에 다음 명령을 입력하세요. ros2 pkg create --build-type ament_cmake --node-name my_node my_package --license Apache-2.0 going to create a new package package name: my_package destination directory: /home/user/ros2_ws/src package format: 3 version: 0.0.0 description: TODO: Package description maintainer: [' '] licenses: ['Apache-2.0'] build type: ament_cmake dependencies: [] node_name: my_node creating folder ./my_package creating ./my_package/package.xml creating source and include folder creating folder ./my_package/src creating folder ./my_package/include/my_package creating ./my_package/CMakeLists.txt creating ./my_package/src/my_node.cpp 새 패키지를 위해 자동 생성된 파일들을 볼 수 있습니다. <2. 패키지 빌드하기> 작업 공간에 패키지를 넣는 것은 여러 패키지를 한 번에 빌드할 수 있기 때문에 특히 유용합니다. 이를 위해 작업 공간의 루트에서 colcon build를 실행합니다. 그렇지 않으면 각 패키지를 개별적으로 빌드해야 합니다. 작업 공간의 루트로 돌아가세요 cd ~/ros2_ws 이제 패키지를 빌드할 수 있습니다. 지난 튜토리얼에서 생성한 ros_tutorials 패키지도 ros2_ws에 있다는 것을 기억하세요. colcon build를 실행하면 turtlesim 패키지도 빌드된다는 것을 알았을 것입니다. 작업 공간에 몇 개의 패키지만 있을 때는 문제가 되지 않지만, 많은 패키지가 있을 때는 colcon build가 오래 걸릴 수 있습니다. 이를 위해 my_package 패키지만 빌드하려면, 다음과 같이 실행할 수 있습니다. colcon build --packages-select my_package <3. 설정파일 소싱하기> 새 패키지와 실행 파일을 사용하려면, 먼저 새 터미널을 열고 메인 ROS 2 설치를 소싱하세요.
