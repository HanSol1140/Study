Nodes 및 Topics 와 같이 이 튜토리얼에서 언급된 일부 개념은 시리즈의 이전 튜토리얼에서 다루었습니다. Turtlesim 패키지가 필요합니다 . 언제나 그렇듯이, 새 터미널을 열 때마다 ROS 2를 소스로 제공하는 것을 잊지 마세요. 새 터미널을 열고 turtlesim과 teleop를 실행합니다. ros2 run turtlesim turtlesim_node ros2 run turtlesim turtle_teleop_key ros2 service list /clear /kill /reset /spawn /teleop_turtle/describe_parameters /teleop_turtle/get_parameter_types /teleop_turtle/get_parameters /teleop_turtle/list_parameters /teleop_turtle/set_parameters /teleop_turtle/set_parameters_atomically /turtle1/set_pen /turtle1/teleport_absolute /turtle1/teleport_relative /turtlesim/describe_parameters /turtlesim/get_parameter_types /turtlesim/get_parameters /turtlesim/list_parameters /turtlesim/set_parameters /turtlesim/set_parameters_atomically 당신은 두 노드가 이름에 파라미터가 포함된 동일한 여섯 개의 서비스를 가지고 있음을 볼 것입니다. 거의 모든 ROS 2 노드는 파라미터가 기반한 이러한 인프라 서비스를 가지고 있습니다. 다음 튜토리얼에서 파라미터에 대해 더 배울 것입니다. 이 튜토리얼에서는 파라미터 서비스를 논의에서 제외하겠습니다. 지금은 /clear, /kill, /reset, /spawn, /turtle1/set_pen, /turtle1/teleport_absolute, 그리고 /turtle1/teleport_relative과 같은 turtlesim 특정 서비스에 초점을 맞추겠습니다. 당신은 Use turtlesim, ros2, and rqt 튜토리얼에서 rqt를 사용하여 이 서비스들과 상호작용하는 것을 기억할 수 있습니다. 서비스는 서비스의 요청과 응답 데이터가 어떻게 구조화되어 있는지를 설명하는 타입을 가지고 있습니다. 서비스 타입은 주제 타입과 비슷하게 정의되지만, 서비스 타입은 두 부분을 가지고 있습니다. : 요청을 위한 한 메시지와 응답을 위한 다른 메시지입니다. 서비스의 타입을 찾으려면 명령을 사용하세요: ros2 service type <서비스_이름> turtlesim의 /clear 서비스를 살펴봅시다. 새 터미널에서 다음 명령을 입력하세요: ros2 service type /clear std_srvs/srv/Empty => Empty 타입은 서비스 호출 시 요청 데이터를 보내지 않고 => 응답을 받을 때도 데이터를 받지 않는다는 것을 의미합니다.
