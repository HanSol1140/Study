HTTPS는 HTTP Secure의 약자 HTTP 프로토콜의 데이터를 암호화하는 방식으로, 웹 사이트의 보안을 강화합니다. SSL(Secure Sockets Layer) 또는 그 후속인 TLS(Transport Layer Security) 프로토콜을 사용하여 통신을 암호화합니다. 정보의 보안 HTTPS를 사용하면, 사용자가 웹사이트에 제공하는 모든 정보가 암호화되어 네트워크 상에서 안전하게 전송됩니다. 이는 특히 신용 카드 번호나 개인 식별 정보와 같은 민감한 데이터를 보호하는데 중요합니다. 데이터 무결성 데이터가 전송 중에 변경되거나 손상되는 것을 방지합니다. 이는 사용자가 보내거나 받는 정보가 중간에 변조되지 않음을 보장합니다. 인증 HTTPS는 웹사이트의 신원을 인증합니다. 이는 사용자가 의도한 사이트와 통신하고 있는지 확인하는데 중요하며, 중간자 공격(man-in-the-middle attack)을 방지합니다. HTTPS의 동작 방식은 다음과 같습니다. 클라이언트(보통 웹 브라우저)는 서버에게 접속을 요청하고, 이를 위해 SSL/TLS 핸드셰이크를 진행합니다. 핸드셰이크 과정에서 서버는 클라이언트에게 공개키가 포함된 인증서를 제공합니다. 클라이언트는 이 공개키를 사용하여 통신에 사용될 대칭키를 암호화하고 서버에게 전달합니다. 서버는 자신만이 소유하고 있는 비공개키로 대칭키를 복호화합니다. 이렇게 해서 서버와 클라이언트는 같은 대칭키를 공유하게 되며, 이를 통해 암호화된 통신이 가능해집니다. 이후 클라이언트와 서버 사이의 모든 데이터 교환은 이 대칭키를 이용해 암호화되어 전송됩니다. HTTPS는 이런 방식으로 웹 통신의 보안을 강화하지만, 처리에 추가적인 계산이 필요하므로 HTTP에 비해 약간의 오버헤드가 발생할 수 있습니다. 그러나 현재의 컴퓨팅 기술 수준에서는 이 오버헤드는 대체로 무시할 수 있을 정도이며, 보안 강화의 이점이 이를 상쇄합니다.
