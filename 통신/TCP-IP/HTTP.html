HTTP는 Hypertext Transfer Protocol의 약자로, 웹상에서 클라이언트와 서버 간에 데이터를 주고받을 수 있는 방법을 제공합니다. HTTP는 웹의 기초를 이루는 기술 중 하나로, 웹 브라우저와 웹 서버가 서로 통신하는데 사용됩니다. HTTP는 TCP/IP 프로토콜 위에서 작동하며, 주로 포트 80을 사용합니다. 클라이언트 (보통 웹 브라우저)는 HTTP를 사용하여 서버에 요청을 보내고, 서버는 이러한 요청에 대한 응답을 반환합니다. 이러한 요청과 응답은 HTTP 메시지라고 불리며, 이 메시지는 요청 라인(request line), 헤더(headers), 그리고 본문(body)으로 구성되어 있습니다. HTTP는 무상태(stateless) 프로토콜입니다. 이 말은, 각 요청이 서버에 의해 별도로 처리되며, 서버는 이전 요청에 대한 정보를 저장하지 않는다는 것을 의미합니다. 이 특성 때문에, 일반적인 HTTP 연결은 한 번의 요청과 응답 후에 바로 종료되는 특성이 있습니다. 예를 들어, 클라이언트가 웹 브라우저를 통해 웹 페이지를 요청하면, 웹 서버는 웹 페이지의 HTML 문서를 HTTP 응답 메시지의 본문 부분에 포함시켜 반환합니다. GET / HTTP/1.1 Host: www.example.com 위의 예시는 클라이언트가 www.example.com의 홈페이지를 요청하는 HTTP 요청 메시지입니다. 이 요청은 GET 메서드를 사용하며, 요청하는 리소스는 "/"(홈페이지)입니다. HTTP는 안전하지 않은 프로토콜이라는 점에서 단점이 있습니다. 전송하는 모든 데이터는 평문으로 전송되기 때문에, 누군가 네트워크를 감시하고 있다면 데이터를 읽을 수 있습니다. 이러한 문제를 해결하기 위해 가 개발되었습니다. HTTPS는 HTTP에 보안 계층을 추가하여, 클라이언트와 서버 간에 주고받는 모든 데이터를 암호화합니다.
