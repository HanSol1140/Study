Philips Semiconductor (지금의 NXP Semiconductors)가 1980년대에 개발한 직렬 통신 프로토콜입니다. 이는 주로 짧은 거리 통신에 사용되며, 저전력 센서, EEPROM, IO 확장기, ADC와 DAC 등과 같은 다양한 저속 주변 장치를 마이크로컨트롤러와 연결하는 데 널리 사용됩니다. I²C는 마스터/슬레이브 구조로 작동합니다. 마스터는 보통 마이크로컨트롤러가 되며, 슬레이브는 센서나 다른 IC들이 됩니다. 이 시스템은 하나의 마스터와 여러 개의 슬레이브를 지원하며, 때로는 여러 마스터를 지원하기도 합니다. I²C 통신은 두 개의 선인 SDA (Serial Data Line) 및 SCL (Serial Clock Line)을 사용하여 데이터를 전송합니다. 모든 장치는 이 두 선에 연결되며, 각 장치는 각각 고유한 주소를 가집니다. 이 주소는 마스터가 특정 슬레이브에게 데이터를 전송하거나 요청할 때 사용됩니다. 특징 및 장단점: 장점 두 개의 선만 필요로 하므로 통신을 위한 하드웨어 인터페이스가 간단합니다. 또한, 주소 지정 방식을 통해 여러 장치를 동일한 버스에 연결할 수 있습니다. 단점 I²C는 짧은 거리 통신에 주로 사용되며, 데이터 전송 속도는 상대적으로 느립니다. 또한, 각 장치에 대해 고유한 주소를 가지고 있어야 하므로 주소 충돌이 발생할 수 있습니다. 적용 사례: 임베디드 시스템, 스마트 폰, TV 등에서 주로 사용됩니다. 센서 데이터를 읽거나 제어 신호를 보내는 등의 작업에 이상적입니다. I²C 통신이 사용되는 코드 예시는 다음과 같습니다 import smbus bus = smbus.SMBus(1) # Raspberry Pi에서 I2C 버스를 사용하도록 설정 DEVICE_ADDRESS = 0x15 # 슬레이브 장치의 I2C 주소 data = bus.read_i2c_block_data(DEVICE_ADDRESS, 0, 16) # 장치에서 16바이트 데이터를 읽음 print(data) 이 예시는 smbus 라이브러리를 사용하여 I²C 통신을 수행하고 있습니다. 슬레이브 장치에서 데이터를 읽어오는 것을 보여주고 있습니다. 코드를 작성할 때 해당 장치의 I²C 주소를 알고 있어야 합니다.
